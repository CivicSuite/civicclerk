CivicClerk

CivicClerk is planned municipal meeting software for agendas, packets, notices, minutes, motions, votes, and public meeting archives.

Status: runtime foundation plus canonical schema, Alembic scaffolding, agenda item lifecycle enforcement, and meeting lifecycle enforcement. IT staff can smoke-check the FastAPI app, inspect the schema plan, and exercise agenda plus meeting transition rules, but packet, notice, vote, minutes, and archive workflows are not implemented yet.

For clerks

Create draft meetings and agenda items through the API today; packet assembly, posting deadlines, and minutes drafting remain planned workflow screens.

For IT

Runtime foundation stack: FastAPI, civiccore==0.2.0, canonical schema metadata, Alembic migration scaffold, agenda lifecycle enforcement, meeting lifecycle enforcement, pytest, docs verification, and placeholder-import CI gates. Planned local services include PostgreSQL 17, Redis 7.2, Celery, Ollama, and nginx-served React.

For residents

Planned public pages will clearly label posted agendas, packets, approved minutes, and archived meeting records.

What ships today

Documentation, governance files, issue templates, discussion seeds, a docs verification gate, a placeholder-import gate, Python package metadata, a FastAPI runtime foundation with root and health endpoints, canonical schema/Alembic scaffolding, agenda item lifecycle enforcement, and meeting lifecycle enforcement with audit entries.
